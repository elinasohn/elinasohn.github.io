For this project..

I wanted to attract the users by placing a simple mini-game using JavaScript on the home page. I searched for examples I could use, and I found this example. I chose the codes because the syntax in this game was simple enough for me to use.

The user starts the game by pressing the play button. Press the button to find the emoji, and when user finds it, the game ends. For each game, the location of emoji is randomly placed.

The game was interesting, but I wanted to add a click animation to this in order to bring more interest to the users. I searched and liked what I found from mo.js that is simple to use. I added the animation to the project.

There is one new challenge in this project. This is my first time using scss to my project. When looking at examples of others in codepen, they were often using mixin and variable using scss. I wanted to use it too. It took a while to understand the syntax, but it was worth it. It was very convenient especially when organizing animations.

What I learned

I searched for various examples to do this project, and I saw many people were writing their codes in scss. In order to use their example, it was necessary to get used to the scss. Their code was very neat and simple, so I thought I wanted to use it too. I looked at the codepen and scss guide page, and it was helpful for me to understand the syntax of the scss.

For example, in many examples, people used the variables to colors and fonts. I followed their ways and It was very convenient! Once I set the color at the beginning of the scss document as var, I could change the color that has the same value easily just changing the value of the var. The nesting function made my coding easy as well. This function is especially useful when I want to add effects such as hover. I set the &:hover selector in the bracket of the element that I want to set the hover, and it made the codes more shorten and easy to read.
